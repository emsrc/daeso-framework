  7  Introduction 
  8  ============ 
  9   
 10  The Munkres module provides an implementation of the Munkres algorithm 
 11  (also called the Hungarian algorithm or the Kuhn-Munkres algorithm), 
 12  useful for solving the Assignment Problem. 
 13   
 14  Assignment Problem 
 15  ================== 
 16   
 17  Let *C* be an *n*\ x\ *n* matrix representing the costs of each of *n* workers 
 18  to perform any of *n* jobs. The assignment problem is to assign jobs to 
 19  workers in a way that minimizes the total cost. Since each worker can perform 
 20  only one job and each job can be assigned to only one worker the assignments 
 21  represent an independent set of the matrix *C*. 
 22   
 23  One way to generate the optimal set is to create all permutations of 
 24  the indexes necessary to traverse the matrix so that no row and column 
 25  are used more than once. For instance, given this matrix (expressed in 
 26  Python): 
 27   
 28  .. python:: 
 29   
 30      matrix = [[5, 9, 1], 
 31                [10, 3, 2], 
 32                [8, 7, 4]] 
 33   
 34  You could use this code to generate the traversal indexes: 
 35   
 36  .. python:: 
 37   
 38      def permute(a, results): 
 39          if len(a) == 1: 
 40              results.insert(len(results), a) 
 41   
 42          else: 
 43              for i in range(0, len(a)): 
 44                  element = a[i] 
 45                  a_copy = [a[j] for j in range(0, len(a)) if j != i] 
 46                  subresults = [] 
 47                  permute(a_copy, subresults) 
 48                  for subresult in subresults: 
 49                      result = [element] + subresult 
 50                      results.insert(len(results), result) 
 51   
 52      results = [] 
 53      permute(range(len(matrix)), results) # [0, 1, 2] for a 3x3 matrix 
 54   
 55  After the call to permute(), the results matrix would look like this:: 
 56   
 57      [[0, 1, 2], 
 58       [0, 2, 1], 
 59       [1, 0, 2], 
 60       [1, 2, 0], 
 61       [2, 0, 1], 
 62       [2, 1, 0]] 
 63   
 64  You could then use that index matrix to loop over the original cost matrix 
 65  and calculate the smallest cost of the combinations: 
 66   
 67  .. python:: 
 68   
 69      n = len(matrix) 
 70      minval = sys.maxint 
 71      for row in range(n): 
 72          cost = 0 
 73          for col in range(n): 
 74              cost += matrix[row][col] 
 75          minval = min(cost, minval) 
 76   
 77      print minval 
 78   
 79  While this approach works fine for small matrices, it does not scale. It 
 80  executes in O(*n*!) time: Calculating the permutations for an *n*\ x\ *n* 
 81  matrix requires *n*! operations. For a 12x12 matrix, that's 479,001,600 
 82  traversals. Even if you could manage to perform each traversal in just one 
 83  millisecond, it would still take more than 133 hours to perform the entire 
 84  traversal. A 20x20 matrix would take 2,432,902,008,176,640,000 operations. At 
 85  an optimistic millisecond per operation, that's more than 77 million years. 
 86   
 87  The Munkres algorithm runs in O(*n*\ ^3) time, rather than O(*n*!). This 
 88  package provides an implementation of that algorithm. 
 89   
 90  This version is based on 
 91  http://www.public.iastate.edu/~ddoty/HungarianAlgorithm.html. 
 92   
 93  This version was written for Python by Brian Clapper from the (Ada) algorithm 
 94  at the above web site. (The ``Algorithm::Munkres`` Perl version, in CPAN, was 
 95  clearly adapted from the same web site.) 
 96   
 97  Usage 
 98  ===== 
 99   
100  Construct a Munkres object: 
101   
102  .. python:: 
103   
104      from munkres import Munkres 
105   
106      m = Munkres() 
107   
108  Then use it to compute the lowest cost assignment from a cost matrix. Here's 
109  a sample program: 
110   
111  .. python:: 
112   
113      from munkres import Munkres, print_matrix 
114   
115      matrix = [[5, 9, 1], 
116                [10, 3, 2], 
117                [8, 7, 4]] 
118      m = Munkres() 
119      indexes = m.compute(matrix) 
120      print_matrix('Lowest cost through this matrix:', matrix) 
121      total = 0 
122      for row, column in indexes: 
123          value = matrix[row][column] 
124          total += value 
125          print '(%d, %d) -> %d' % (row, column, value) 
126      print 'total cost: %d' % total 
127   
128  Running that program produces:: 
129   
130      Lowest cost through this matrix: 
131      [5, 9, 1] 
132      [10, 3, 2] 
133      [8, 7, 4] 
134      (0, 0) -> 5 
135      (1, 1) -> 3 
136      (2, 2) -> 4 
137      total cost=12 
138   
139  The instantiated Munkres object can be used multiple times on different 
140  matrices. 
141   
142  Non-square Cost Matrices 
143  ======================== 
144   
145  The Munkres algorithm assumes that the cost matrix is square. However, it's 
146  possible to use a rectangular matrix if you first pad it with 0 values to make 
147  it square. This module automatically pads rectangular cost matrices to make 
148  them square. 
149   
150  Notes: 
151   
152  - The module operates on a *copy* of the caller's matrix, so any padding will 
153    not be seen by the caller. 
154  - The cost matrix must be rectangular or square. An irregular matrix will 
155    *not* work. 
156   
157  Calculating Profit, Rather than Cost 
158  ==================================== 
159   
160  The cost matrix is just that: A cost matrix. The Munkres algorithm finds 
161  the combination of elements (one from each row and column) that results in 
162  the smallest cost. It's also possible to use the algorithm to maximize 
163  profit. To do that, however, you have to convert your profit matrix to a 
164  cost matrix. The simplest way to do that is to subtract all elements from a 
165  large value. For example: 
166   
167  .. python:: 
168   
169      from munkres import Munkres, print_matrix 
170   
171      matrix = [[5, 9, 1], 
172                [10, 3, 2], 
173                [8, 7, 4]] 
174      cost_matrix = [] 
175      for row in matrix: 
176          cost_row = [] 
177          for col in row: 
178              cost_row += [sys.maxint - col] 
179          cost_matrix += [cost_row] 
180   
181      m = Munkres() 
182      indexes = m.compute(cost_matrix) 
183      print_matrix('Lowest cost through this matrix:', matrix) 
184      total = 0 
185      for row, column in indexes: 
186          value = matrix[row][column] 
187          total += value 
188          print '(%d, %d) -> %d' % (row, column, value) 
189   
190      print 'total profit=%d' % total 
191   
192  Running that program produces:: 
193   
194      Highest profit through this matrix: 
195      [5, 9, 1] 
196      [10, 3, 2] 
197      [8, 7, 4] 
198      (0, 1) -> 9 
199      (1, 0) -> 10 
200      (2, 2) -> 4 
201      total profit=23 
202   
203  The ``munkres`` module provides a convenience method for creating a cost 
204  matrix from a profit matrix. Since it doesn't know whether the matrix contains 
205  floating point numbers, decimals, or integers, you have to provide the 
206  conversion function; but the convenience method takes care of the actual 
207  creation of the cost matrix: 
208   
209  .. python:: 
210   
211      import munkres 
212   
213      cost_matrix = munkres.make_cost_matrix(matrix, 
214                                             lambda cost: sys.maxint - cost) 
215   
216  So, the above profit-calculation program can be recast as: 
217   
218  .. python:: 
219   
220      from munkres import Munkres, print_matrix, make_cost_matrix 
221   
222      matrix = [[5, 9, 1], 
223                [10, 3, 2], 
224                [8, 7, 4]] 
225      cost_matrix = make_cost_matrix(matrix, lambda cost: sys.maxint - cost) 
226      m = Munkres() 
227      indexes = m.compute(cost_matrix) 
228      print_matrix('Lowest cost through this matrix:', matrix) 
229      total = 0 
230      for row, column in indexes: 
231          value = matrix[row][column] 
232          total += value 
233          print '(%d, %d) -> %d' % (row, column, value) 
234      print 'total profit=%d' % total
