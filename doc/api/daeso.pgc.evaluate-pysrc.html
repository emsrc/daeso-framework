 20  """ 
 21  Evaluation of alignment by comparing a corpus containing predicted alignments with 
 22  a reference corpus containing true alignments. 
 23   
 24   
 25  EVALUATION OF ALIGNMENT 
 26   
 27  An (unlabeled) alignment consists of a set of node pairs <n,m> where n is a 
 28  node of the source tree and m is a noe of the target tree. Given a set of true 
 29  and predicted alignments 
 30   
 31      A_true = { <n1,m1>, <n2,m2>, ... } 
 32      A_pred = { <n1',m1'>, <n2',m2'>, ... } 
 33   
 34  the recall, precision and F-score are defined as  
 35   
 36   
 37                | intersection(A_true, A_pred) | 
 38      recall = ---------------------------------- 
 39                          | A_true |                 
 40   
 41   
 42                           
 43                   | intersection(A_true, A_pred) | 
 44      precision = ---------------------------------- 
 45                           | A_pred |    
 46   
 47   
 48                2 
 49           (beta  + 1) * prec * rec 
 50      F =  ------------------------ 
 51                  2  
 52              beta  * prec + rec 
 53               
 54               
 55  where 0 <= beta <= 1. 
 56   
 57   
 58  As a shorthand, the following three counts are used: 
 59   
 60      True ;= | A_true | 
 61      Pred := | A_pred | 
 62      Common := | intersection(A_true, A_pred) | 
 63       
 64  hence recall=Common/True and precision=Common/Pred.  
 65       
 66  Whenever True or Pred is zero, the recall and precision respectively are 
 67  technically undefined. For convenience, however, the implementation returns 
 68  zero. The same goes for the F score when both precision and recall equals 
 69  zero. 
 70   
 71   
 72  EVALUATION OF LABELED ALIGNMENT 
 73   
 74  A labeled alignment consists of a set of tuples <n,m, l> where n is a node of 
 75  the source tree, m is a node of the target tree, and l is the relation label. 
 76  Given a set of true and predicted alignments 
 77   
 78      A_true = { <n1,m1, l1>, <n2,m2,l2>, ... } 
 79      A_pred = { <n1',m1',l1'>, <n2',m2',l2'>, ... } 
 80   
 81  the micro recall, precision and F-score are defined the same as above. 
 82   
 83  The score on a particular label is calculated by first restrictng A_true and 
 84  A_pred to only those tuples with that label, disregarding all tuples with 
 85  other labels, and then again calculating precision, recall and F score in the 
 86  above way. 
 87   
 88  The overall macro recall, precision and F-score are calculating by taking the 
 89  average over the scores on each relation label. In contrast to the micro 
 90  scores, the macro scores ignores the fact that the distribution of the labels 
 91  may be skewed. 
 92   
 93  MULTIPLE CORPORA 
 94   
 95  For multiple corpora, first counts and scores are calculated for each separate 
 96  corpus, and then average scores are calculated. The macro avarage is the 
 97  avarage over all separate score. The micro avarage is calculated by first 
 98  summing the counts of True, Pred and Common respectively, and then computing 
 99  precision, recall and F-score over these summed counts.
