  56      """ 
  57      The ParallelGraphCorpus class represents a corpus of parallel graphs as a 
  58      list of GraphPair objects.  A corpus can be read from and 
  59      written to a custom XML format. 
  60       
  61      Graph pairs and graphbanks 
  62      --------------------------- 
  63       
  64      Each GraphPair consist of a source and a target graph pair, which are 
  65      defined in separate graphbanks, plus a number of labeled node alignments. 
  66      See the GraphPair class for details. Although the corpus file specifies 
  67      the file paths for all the graphbanks providing source and target graphs, 
  68      once a corpus file is read (parsed and processed), a ParallelGraphCorpus 
  69      object does not contain any direct references to either graphs or graph 
  70      banks. All references to graphs and graphbanks are kept in GraphPair 
  71      objects. This avoids the need for a complicated administration which keeps 
  72      track of which graphbanks are added (or have become redundant) after 
  73      inserting or deleting graphs pairs. Moreover, graphbanks which are no 
  74      longer in use by any graph pair are automatically garbage collected. Only 
  75      when the corpus needs to be written to file, all graphbanks referred to 
  76      are collected, and their file paths are included in the ouput. 
  77       
  78      By default reading a corpus from file will automatically retrieve the 
  79      required graphs from the graphbanks (cf. SparseTreeBank class). However, 
  80      a corpus can be opened without really opening and parsing the graphbanks, 
  81      which saves memory and time (cf. GraphStub class). This is usefull in 
  82      cases where access to graph nodes is not required. 
  83       
  84      Relations 
  85      --------- 
  86       
  87      The node alignments of pair of graphs are labeled according to a list of 
  88      of alignment relations (stored in self._relations). The list of possible 
  89      relations is fixed for all graph pairs in a corpus. It is initialized only 
  90      when creating/reading a corpus, and must not be changed afterwards. 
  91       
  92      Meta-data 
  93      --------- 
  94       
  95      The XML file format reserves a particular node in the XML tree (i.e 
  96      <corpus_meta_data>) for arbitrary user-defined meta-data regarding the 
  97      corpus. This information is stored as Element object (cf. 
  98      xml.etree.Elementree).
